# Fandqi — Central UI Cleanup & Audit Report

## الهدف
تنظيم واجهة المشروع حول Design System مركزي واحد، بحيث تبقى كل التعديلات القادمة تراكمية وقابلة للتطوير بدون كسر التدرجات أو الهوفر أو أحجام الأزرار أو شكل الكروت.

## ما تم تنفيذه

### 1. تنظيف CSS من الاستثناءات غير المركزية
- إزالة استخدام `!important` من ملف `app.css` حتى لا تصبح التعديلات القادمة معتمدة على كسر الأولويات.
- الحفاظ على نفس النتيجة البصرية عبر ترتيب الطبقات وزيادة التحديد المنطقي بدل الإجبار.

### 2. منع الألوان الصلبة خارج التوكنز
- إزالة الألوان المكتوبة مباشرة داخل `app.css` مثل HEX و RGB/RGBA.
- نقل/ربط الألوان النهائية عبر متغيرات مركزية داخل `tokens.css`.
- أصبح تعديل لون أساسي أو لون زر يتم من التوكنز بدل البحث داخل كل صفحة.

### 3. تثبيت طبقات التصميم الأساسية
تم الحفاظ على الطبقات التالية كمرجع واجهة ثابت:
- خلفية التطبيق العامة `layout-bg`.
- تدرج `Sidebar`.
- تدرج `Topbar`.
- خلفية منطقة المحتوى.
- خلفية `Page Shell`.

### 4. حماية المكونات المركزية
تم التأكد أن هذه المكونات ما زالت مربوطة بالنظام المركزي:
- Buttons: `.btn`, `.ds-btn`
- Inputs/Selects/Textareas: `.input`, `.select`, `.textarea`, `.ds-control`
- Cards: `.ds-card`, `.room-card`, `.staff-card`, `.reservation-card`
- Filters: `.ds-filters`
- Tabs: `.ds-tab-btn`, `.settings-tab-btn`, `.checkio-tab`
- Modals: `.ds-modal-card`
- Badges/Chips: `.ds-badge`

### 5. حماية التعديلات السابقة
تم الحفاظ على الإصلاحات السابقة:
- Hover زر الحذف.
- Hover تبويبات الإعدادات.
- Hover تبويبات الدخول والمغادرة.
- زر العين.
- التدرجات اللونية للسايدبار والتوب بار والمحتوى.
- ثبات أحجام الأزرار بعد تغيير الحالة.
- توحيد ارتفاع كروت الحجوزات.
- وضوح عناصر طلبات المطعم والكافتريا.
- أرشيف الغرف واستعادة الغرف.
- توحيد كروت الباقات.
- إصلاح `formatDateTime`.

## قواعد إلزامية لأي تعديل قادم

1. لا تبنِ زرًا أو كرتًا أو حقلًا داخل صفحة بشكل منفصل إذا كان له صنف مركزي.
2. لا تستخدم `!important` إلا إذا تم توثيق سبب استثنائي جدًا، والأصل ممنوع.
3. لا تستخدم ألوان HEX/RGB داخل `app.css`؛ أي لون جديد يضاف أولًا إلى `tokens.css`.
4. أي عنصر يتولد ديناميكيًا يجب أن يمر عبر `applyCentralDesignSystem`.
5. عند إصلاح صفحة واحدة، لا يجوز الرجوع لنسخة قديمة لأن ذلك يحذف إصلاحات متراكمة.
6. قبل التسليم يجب تشغيل `npm run quality:full`.
