# Phase 90 — Stable UI Recovery Baseline

## الهدف

اعتماد نسخة Phase 87 كأكثر نسخة مستقرة بصريًا بعد ظهور انحدارات واجهة خطيرة في محاولات المركزية الهجومية اللاحقة.

## قرار الإغلاق

تم الرجوع إلى قاعدة Phase 87 المستقرة بصريًا وعدم البناء فوق الطبقات التي سببت تداخل الكروت والعناصر والفراغات غير المدروسة.

## ما تم الحفاظ عليه

- نظام الأزرار المركزي مع الأيقونات.
- تنوع ألوان الأزرار حسب معنى الإجراء.
- إغلاق اللغة المركزي حتى Phase 86.
- إغلاق التجاوب حتى Phase 85.
- واجهات الموظفين وصاحب الفندق وصاحب المنصة المستقرة حتى Phase 87.
- كل منطق العمل والتخزين المحلي والصلاحيات ومسارات البيانات.

## ما تم استبعاده من خط الأساس الجديد

- القواعد المركزية الهجومية التي أدت إلى كروت داخل كروت.
- التعميمات التي أثرت على الغرف والموظفين والمطعم والصيانة والمدفوعات والتقارير والإشعارات.
- أي طبقة Design System تفرض شكلًا عامًا على عناصر تحتاج سياقًا خاصًا.

## الخطة بعد الرجوع

أي محاولة مركزية لاحقة يجب أن تكون آمنة وتدريجية:

1. صفحة واحدة أو مجموعة صغيرة فقط في كل مرة.
2. فحص بصري بعد كل تعديل.
3. عدم استخدام selectors عامة تضرب كل الكروت أو كل البانلات.
4. الحفاظ على Phase 87 كمرجع بصري ثابت.
5. عدم تسليم أي مركزية جديدة قبل التأكد من عدم ظهور تداخل أو قص أو فراغات عشوائية.

## الفحص

تم تشغيل الفحص الكامل بعد اعتماد خط الأساس المستقر.
