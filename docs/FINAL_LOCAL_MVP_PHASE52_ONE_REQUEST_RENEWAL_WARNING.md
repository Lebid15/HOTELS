# Final Local MVP Closure — Phase 52

## One Request Per Package + Renewal Warning

تم تطبيق شرط إرسال طلب الباقة مرة واحدة فقط، وإضافة تحذير التجديد قبل انتهاء الاشتراك بثلاثة أيام.

## منع تكرار الطلب

- عند طلب تغيير باقة، يتم تسجيل الطلب مرة واحدة فقط.
- إذا كان الطلب موجودًا لنفس الفندق ونفس الباقة، لا يمكن إرسال طلب جديد.
- يظهر زر الباقة بحالة معطلة مع نص: تم إرسال الطلب.
- الحماية موجودة في الواجهة وفي منطق `createManagerSubscriptionRequest`.

## تجديد الباقة قبل 3 أيام

عندما يكون الاشتراك فعالًا وباقي 3 أيام أو أقل على الانتهاء:

- يظهر تحذير داخل قسم الباقة المفعّلة.
- يظهر زر: تجديد الآن.
- إذا تم إرسال طلب تجديد سابقًا، يصبح الزر معطلًا.
