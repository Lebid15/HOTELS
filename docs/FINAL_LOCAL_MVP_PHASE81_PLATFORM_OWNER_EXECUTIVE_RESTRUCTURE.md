# Fandqi Phase 81 — Platform Owner Executive Restructure

## الهدف
إعادة هيكلة واجهة صاحب المنصة لتصبح مركز قيادة احترافي متناسق مع واجهات صاحب الفندق التي تم تحسينها في المراحل السابقة، مع الابتعاد عن الاعتماد الجدولي وتحويلها إلى كروت تشغيلية واضحة.

## نطاق التنفيذ
- لوحة التحكم الرئيسية لصاحب المنصة.
- صفحة الفنادق.
- صفحة مدراء الفنادق.
- صفحة الباقات.
- صفحة الاشتراكات.
- صفحة طلبات الاشتراك.
- نافذة ملف الفندق من منظور صاحب المنصة.

## أهم التعديلات
1. بناء Executive Dashboard جديد يعرض مؤشرات الفنادق، الاشتراكات، الطلبات، الإيرادات، والتنبيهات التشغيلية.
2. تحويل الفنادق إلى كروت عميل متقدمة تعرض حالة الاشتراك، المدير، الغرف، الموظفين، الحجوزات، والمهام المفتوحة.
3. إضافة نافذة ملف فندق احترافية تعرض بيانات الفندق والمدير والاشتراك والدفعات.
4. تحسين كروت الاشتراكات مع مؤشر مدة الاشتراك وحالة الدفع والأزرار التشغيلية.
5. تحسين صفحة طلبات الاشتراك كصندوق معاملات مستقل مع فلاتر حالة واضحة.
6. توحيد الهيدر والفلاتر والكروت والمسافات لتتوافق مع Design System المركزي.

## ما لم يتغير
- لم يتم إضافة Backend.
- لم يتم تغيير localStorage.
- لم يتم تغيير صلاحيات المستخدمين.
- لم يتم كسر منطق Phase 80 للباقة التجريبية التلقائية وطلبات الاشتراك المنفصلة.
