# خريطة Backend مستقبلية لمشروع فندقي

هذه الخريطة تحافظ على نفس الواجهات الحالية، لكنها تنقل التخزين والمنطق من `localStorage` إلى Backend حقيقي لاحقًا.

## الكيانات الأساسية

- PlatformOwner
- Hotel
- HotelManager
- StaffUser
- Room
- Reservation
- Guest
- FoodMenuItem
- FoodOrder
- Payment
- AccountStatement
- HousekeepingTask
- MaintenanceTicket
- ReportSnapshot
- AuditLog

## العلاقات المهمة

- كل فندق يملك غرفًا وموظفين وإعدادات.
- كل حجز مرتبط بفندق وغرفة ونزيل رئيسي ومرافقين.
- كل طلب مطعم يمكن أن يكون نقدي، إلكتروني، أو على حساب غرفة.
- كل طلب على حساب غرفة يجب أن يدخل في كشف حساب الحجز ويمنع الخروج عند وجود متبقي.
- كل خروج يحول الغرفة إلى تنظيف، ولا تصبح متاحة إلا بعد تأكيد التنظيف.
- الصيانة تعيد الغرفة إلى التنظيف بعد الإنجاز وليس إلى متاحة مباشرة.

## أولويات النقل إلى Backend

1. Auth حقيقي وصلاحيات حسب الدور.
2. قاعدة بيانات PostgreSQL.
3. API للفنادق والغرف والحجوزات.
4. API للنزلاء والدخول والمغادرة.
5. API للمطعم والكافتريا والحسابات.
6. API للتنظيف والصيانة.
7. التقارير والاستعلامات المالية.
8. سجل تدقيق AuditLog لكل عملية حساسة.
9. تخزين ملفات وصور عبر Object Storage.
10. نسخ احتياطي دوري على مستوى قاعدة البيانات.
