# Phase 86 — Central i18n Language Closure

## الهدف
إغلاق ملف اللغة بشكل مركزي بعد Phase 85 حتى تكون واجهات المشروع مغطاة بالعربية والإنجليزية من مصدر واحد قدر الإمكان، مع منع تكرار مشكلة نقص مفاتيح اللغة في المراحل اللاحقة.

## ما تم تنفيذه

- تمت مراجعة كل استدعاءات `t()` داخل ملفات JavaScript.
- تمت إضافة كل المفاتيح الناقصة إلى:
  - `apps/web/public/locales/ar.json`
  - `apps/web/public/locales/en.json`
- أصبح عدد مفاتيح اللغة متطابقًا بين العربية والإنجليزية.
- تمت تغطية مفاتيح المراحل الحديثة:
  - صفحات تسجيل الدخول / إنشاء الحساب / استعادة كلمة المرور.
  - إشعارات التوب بار والإشعارات التشغيلية.
  - طلبات الاشتراك لصاحب المنصة.
  - واجهات صاحب المنصة التنفيذية.
  - واجهات الموظفين ومركز الاستقبال.
  - تفاصيل الحجز والفلاتر.
- تم توسيع i18n ليحدث عنوان المتصفح وشاشة التحميل حسب اللغة.
- تم نقل نصوص ظاهرة متبقية إلى المركزية مثل حرف الشعار وأسماء اللغات وفواصل القوائم ورسالة منع نافذة الطباعة.

## فحص جديد
تمت إضافة سكربت:

```bash
npm run i18n:closure-audit
```

ويتحقق من:

- وجود ملفي `ar.json` و `en.json`.
- تطابق كامل للمفاتيح بين اللغتين.
- عدم وجود قيم فارغة.
- تغطية كل مفاتيح `t()` الحرفية داخل ملفات JavaScript.

كما تم إدخال الفحص داخل:

```bash
npm run quality:full
```

## النتيجة

- `ar.json`: 1529 مفتاح.
- `en.json`: 1529 مفتاح.
- كل مفاتيح `t()` الحرفية مغطاة.
- فحص اللغة نجح.
- الفحص الكامل نجح.

## ملاحظة مهمة
بعض النصوص داخل الكود تبقى كقيم تشغيلية أو fallback آمن أو كلمات تصنيف داخل نظام الأيقونات وليست نصوص واجهة مباشرة. أما النصوص الظاهرة الأساسية فقد أصبحت مربوطة بالمركزية.
