# Fandqi Phase 99 — Staff Archived Restore Action

## الهدف
تصحيح تجربة الموظفين المؤرشفين داخل صفحة الموظفين بعد إضافة فلتر المؤرشفين في Phase 98.

## المشكلة
عند عرض الموظفين المؤرشفين كان كرت الموظف لا يزال يعرض زر **أرشفة**، وهذا غير منطقي لأن الموظف مؤرشف بالفعل. الإجراء الصحيح هو **استعادة**.

## التعديل
- إضافة دعم إجراء `restore-staff`.
- إضافة أيقونة `refreshCw` لإجراء الاستعادة.
- جعل زر الموظف المؤرشف يظهر باسم **استعادة** وبتدرج نجاح.
- تحديث حالة الموظف إلى `active` عند الضغط على الاستعادة.
- تحديث واجهة الموظفين مباشرة بعد الاستعادة.
- إضافة مفتاح اللغة `staff.actions.restore` في `ar.json` و `en.json`.

## النطاق
التعديل محصور في صفحة الموظفين فقط، ولا يمس بقية صفحات المشروع.

## الملفات المتأثرة
- `apps/web/public/assets/js/modules/05-staff.js`
- `apps/web/public/assets/js/modules/06-rooms-dashboard.js`
- `apps/web/public/locales/ar.json`
- `apps/web/public/locales/en.json`
- `FANDQI_CHANGE_NOTES.md`

## نتيجة الإغلاق
أصبحت حالة المؤرشفين أكثر وضوحًا: عند عرض موظف مؤرشف تظهر إجراءات مناسبة تشغيليًا بدل تكرار زر الأرشفة.
