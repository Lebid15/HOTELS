# Phase 87 — Central UI System Closure

## الهدف
إغلاق مركزية عناصر التصميم في فندقي بعد إغلاق ملف اللغة، بحيث لا تكون كل صفحة بأسلوب خاص، بل يعمل المشروع بروح تصميم واحدة عبر صاحب المنصة، صاحب الفندق، الموظفين، المصادقة، الإشعارات، التقارير، النماذج، والمودالات.

## العناصر التي تم جعلها مركزية

- الأزرار بأنواعها وألوانها.
- الأيقونات داخل الأزرار.
- الكروت والبانلات.
- الشارات والحالات.
- الحقول والنماذج.
- الفلاتر وأشرطة البحث.
- التبويبات.
- المودالات.
- الجداول ومناطق التمرير.
- عناصر صاحب المنصة التنفيذية.
- عناصر الموظفين ومركز الاستقبال.
- أزرار الموبايل والتوب بار.

## نظام الأزرار
تم الحفاظ على تنوع ألوان الأزرار بدل جعلها لونًا واحدًا:

- `primary` للإضافة، الحفظ، الدخول، التأكيد، والتفعيل.
- `accent` للعرض، الفتح، التفاصيل، المعاينة، والإعداد.
- `success` للتجديد، الدفع، الاستعادة، والتحديثات الناجحة.
- `luxury` للطباعة، الفواتير، الإيصالات، والتصدير.
- `warning` للصيانة، التنظيف، الإيقاف المؤقت، المتابعة، والفلترة.
- `danger` للحذف، الرفض، الإلغاء، الأرشفة، والتعطيل.
- `neutral` للأزرار العامة مثل اللغة، الإشعارات، والتحديث.

## الأيقونات داخل الأزرار
تمت إضافة طبقة JavaScript مركزية تقوم بفحص كل زر بعد رسم الصفحة. إذا كان الزر نصيًا ولا يحتوي على أيقونة، يتم توليد أيقونة مناسبة له تلقائيًا حسب:

- `data-action`
- `data-owner-modal`
- `data-owner-action`
- `data-manager-dashboard-action`
- النص الظاهر
- `title`
- `aria-label`

هذا يحافظ على الأزرار القديمة ويمنع تكرار تعديل كل صفحة يدويًا.


وقد تم إدخاله داخل:

```bash

```

## مبدأ الحماية
لم يتم تغيير منطق البيانات أو التخزين المحلي أو الصلاحيات. المرحلة هي إغلاق تصميمي مركزي، وليست توسعة وظيفية جديدة.
