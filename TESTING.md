# اختبارات funduqii


> **حالة العدّ:** عدد الاختبارات الحالي **يُثبَّت في المرحلة 11** عند تشغيل الحزمة الكاملة.
> آخر رقم موثَّق كان ~**152** اختبارًا (من مراحل الإغلاق التجاري)، **يجب تأكيده**، وهو **لا يشمل** اختبارات سلسلة التحصين (المراحل 1–8) المؤجّلة أدناه. لا يُدّعى نجاح الاختبارات النهائية قبل تشغيلها فعلًا.

## ب) الاختبارات المؤجّلة إلى المرحلة 11 (سلسلة التحصين 1–8)

> هذه هي حزمة الاختبارات التي **يجب كتابتها وتشغيلها في المرحلة 11** لتثبيت جاهزية سلسلة التحصين. حتى تُكتب وتُشغَّل، تبقى هذه المراحل «كود مُغلق · اختبار مؤجّل».

### 1) FolioCharge (المرحلة 1)
- [ ] إنشاء FolioCharge.
- [ ] إبطال FolioCharge بسبب → ينجح.
- [ ] رفض الإبطال بدون سبب (400).
- [ ] رفض الإبطال المكرّر (400).
- [ ] منع تعديل FolioCharge مبطل (400).
- [ ] استثناء المبطل من `balance_due` / الرصيد المستحق.
- [ ] تسجيل حدث `folio_charge.void` في Audit Log.
- [ ] منع `DELETE` (405 — لا مسار حذف).

### 2) بوّابة الحجز العام (المرحلة 2)
- [ ] فندق مخفي (`public_listing_enabled=false`) لا يظهر ولا يقبل حجزًا.
- [ ] فندق بلا قبول اتفاقية فعّالة لا يقبل حجزًا.
- [ ] فندق باشتراك منتهٍ لا يقبل حجزًا.
- [ ] باقة لا تسمح بالظهور → لا يظهر.
- [ ] باقة لا تسمح بالحجز → لا يقبل حجزًا.
- [ ] غرفة غير ظاهرة (`show_in_public=false`) لا تظهر في التوفّر.
- [ ] تجاوز `hotel_id` مباشرةً لفندق غير مؤهّل → يُرفَض (400).
- [ ] `availability` لفندق لا يقبل الحجز → قائمة فارغة.

### 3) manage_token (المرحلة 3)
- [ ] إنشاء حجز عام يُرجع `manage_token`.
- [ ] إنشاء حجز عام يُرجع `manage_url`.
- [ ] lookup برقم + هاتف **لا** يُرجع `manage_token`.
- [ ] lookup برقم + هاتف **لا** يُرجع `manage_url`.
- [ ] lookup برقم + token **لا** يُرجع `manage_token`.
- [ ] إلغاء حجز عبر `token` ينجح.
- [ ] رسالة الفشل عامّة (لا تكشف أيّ الحقلين خاطئ).

### 4) Payment (المرحلة 4)
- [ ] دفعة مفردة صحيحة (amount + method).
- [ ] دفعة مقسّمة صحيحة (المجموع = amount).
- [ ] رفض مجموع أقل من amount.
- [ ] رفض مجموع أكبر من amount.
- [ ] رفض قيمة سالبة.
- [ ] رفض `amount = 0`.
- [ ] `settle_and_checkout` يُنشئ دفعة متّسقة (amount = مجموع الأجزاء).
- [ ] الدفعة المبطلة لا تدخل في الإجمالي.

### 5) تطبيع الهاتف (المرحلة 5)
- [ ] حجز محفوظ بصيغة يُعثر عليه بصيغة أخرى (+/00/0).
- [ ] أرقام بمسافات/شرطات تعمل.
- [ ] أرقام عربية/فارسية تُحوَّل وتُطابِق.
- [ ] رقم خاطئ لا يُرجع حجزًا.
- [ ] Public lookup **لا** يكشف `guest_phone_normalized`.
- [ ] Public lookup يُبقي الهاتف **مقنّعًا**.
- [ ] مسار `manage_token` لم يتأثّر.
- [ ] الـbackfill يعمل على البيانات القديمة.

### 6) الصلاحيات (المرحلة 6)
- [ ] مستخدم بلا `UserProfile` → 403 من قسم محميّ.
- [ ] `username=manager` بلا Profile → لا دور.
- [ ] `username=platform` بلا Profile → لا دور.
- [ ] `username=reception` بلا Profile → لا دور.
- [ ] `UserProfile.role=manager` يعمل.
- [ ] `UserProfile.role=reception` يعمل.
- [ ] `UserProfile.role=platform_owner` يعمل.

### 7) Throttling (المرحلة 7)
- [ ] `availability` → 429 بعد تجاوز الحد (public_availability 30/min).
- [ ] `public booking` → 429 بعد تجاوز الحد (5/min).
- [ ] `lookup/manage` → 429 بعد تجاوز الحد (20/min).
- [ ] `cancel` → 429 بعد تجاوز الحد (10/min).
- [ ] `review` → 429 بعد تجاوز الحد (5/min).
- [ ] الاستخدام الطبيعي (عدد معقول/دقيقة) لا يتأثّر.

### 8) FoodOrder (المرحلة 8)
- [ ] طلب بدفع نقدي كامل.
- [ ] طلب بدفع إلكتروني كامل.
- [ ] طلب ببطاقة كامل.
- [ ] طلب على حساب الغرفة (`amount_room`).
- [ ] طلب مقسّم صحيح (المجموع = amount).
- [ ] رفض مجموع أقل من الإجمالي.
- [ ] رفض مجموع أكبر من الإجمالي.
- [ ] رفض قيمة سالبة.
- [ ] تعديل الأصناف بلا إرسال توزيع لا يكسر الطلب.
- [ ] PATCH بتوزيع خاطئ يُرفَض.
- [ ] التقارير (تجمع على `amount`/`payment_method`) لا تتأثّر.
