# RELEASE_CHECKLIST — قائمة التحقّق قبل النشر الإنتاجي (funduqii)

> تُملأ خاناتها فعليًّا قبل كل إصدار إنتاجي. لا يُنشَر ما لم تكتمل الخانات الإلزامية.

## 1) بوّابة الجودة (يجب أن تمرّ كلها)
- [ ] `cd backend && python manage.py test api` — كل الاختبارات خضراء (حاليًا **119**).
- [ ] `python manage.py check` — 0 مشاكل.
- [ ] `python manage.py makemigrations --check --dry-run` — لا ترحيلات معلّقة.
- [ ] `python manage.py check --deploy` — نظيف **بمتغيّرات بيئة الإنتاج الحقيقية** (لا تحذيرات W004/W008/W009/W012/W016/W018). *(م9: الإعدادات جاهزة — يعطي «0 issues» بالأمر الكامل في `DEPLOY.md §2.2`؛ يُعاد التحقّق بالدومين/المفتاح الحقيقيّين عند الإصدار.)*
- [ ] `cd frontend && npm run lint` — exit 0.
- [ ] `npm run typecheck` — نظيف.
- [ ] `npm run build` — ناجح (بلا اعتماد شبكة — الخط محلّي عبر `next/font/local`).
- [ ] `npm audit` — لا ثغرات عالية/حرجة (المتوسطة موثّقة ومقبولة أو مُعالَجة).

## 2) البيئة والأسرار
- [ ] `DEBUG=false`.
- [ ] `SECRET_KEY` قويّ (لا الافتراضي) — الإقلاع يفشل بدونه في الإنتاج.
- [ ] `ALLOWED_HOSTS` مضبوط للدومين.
- [ ] `CSRF_TRUSTED_ORIGINS` مضبوط.
- [ ] `DATABASE_URL` يشير إلى PostgreSQL الإنتاجي.
- [ ] `NEXT_PUBLIC_API_URL` (في `frontend/.env`) يشير إلى API الإنتاج.
- [ ] لا أسرار في الكود أو في Git.

## 3) قاعدة البيانات والملفات
- [ ] `python manage.py migrate` مُطبَّق على قاعدة الإنتاج.
- [ ] `python manage.py collectstatic` (WhiteNoise مفعّل).
- [ ] `python manage.py create_platform_owner` لإنشاء حساب المالك الأوّل بأمان.
- [ ] سياسة تخزين الوسائط/الوثائق محدّدة.

## 4) الأمان
- [ ] HTTPS + HSTS + Secure cookies مفعّلة (عبر البيئة).
- [ ] Throttling فعّال على: الدخول، الحجز العام، إدارة الحجز، التوفّر، التقييمات.
- [ ] عزل المستأجرين مُختبَر (لا وصول عبر الفنادق).
- [ ] لا حذف مالي مباشر (إبطال + Audit Log) — مُطبَّق.
- [ ] الحجز العام محمي (رقم عشوائي + رمز إدارة + منع تصادم).

## 5) بعد النشر (Smoke test)
- [ ] تسجيل دخول مالك المنصّة يعمل.
- [ ] إنشاء فندق + باقة + اشتراك.
- [ ] ظهور الفندق في الموقع العام (بعد استيفاء الشروط).
- [ ] حجز عام ناجح + رقم/رمز إدارة.
- [ ] تسجيل دخول/دفع/خروج + إغلاق يومي.
- [ ] لا أخطاء 500 في السجلّات.

## 6) عمليات (مُوصى قبل التوسّع)
- [ ] نسخ احتياطي مجدوَل + اختبار استعادة. *(الإجراء موثّق في `DEPLOY.md §7`)*
- [ ] مراقبة (uptime/errors) + تنبيهات. *(الإجراء موثّق في `DEPLOY.md §7`)*
- [ ] خطة Rollback موثّقة. *(موثّقة في `DEPLOY.md §7`)*
- [x] بوّابة CI تمنع النشر عند فشل أي فحص من القسم (1). *(`.github/workflows/ci.yml` — تُفعَّل كفحص إلزامي في حماية الفرع)*
