# Fandqi Change Notes — ملاحظات حماية التعديلات المتتالية

هذا الملف إلزامي قبل أي تعديل جديد على المشروع. الهدف منه منع الرجوع للخلف أو حذف تعديل سابق بدون قصد.

## قواعد ثابتة لا تُكسر

1. التعديلات يجب أن تكون **متتالية وتراكمية**، ولا يجوز الرجوع لنسخة أقدم ثم البناء عليها إذا كانت لا تحتوي آخر الإصلاحات.
2. لا يتم حذف أي تنسيق سابق تم اعتماده إلا إذا طلب المستخدم ذلك صراحة.
3. الخلفيات المتدرجة للـ **Sidebar** والـ **Topbar** وخلفية المحتوى يجب أن تبقى موجودة ولا تُستبدل بخلفية بيضاء/مسطحة.
4. تعديل لون زر أو Hover يجب أن يكون محددًا للعنصر المطلوب فقط، ولا يجوز أن يغير الخلفيات العامة أو نظام الألوان الكامل.
5. أي زر يتم إصلاحه يجب إصلاحه مركزيًا عند الحاجة، لكن بدون التأثير على باقي المكونات.

## آخر التعديلات المحمية

### 1. زر حذف نوع الغرفة
- تم إصلاح خلفية زر الحذف عند الوقوف عليه حتى لا تختفي الأيقونة الحمراء.
- المطلوب كان تغيير **خلفية الزر** وليس لون الخط/الأيقونة.

### 2. تبويبات إعدادات الفندق
- تم إصلاح خلفية Hover لتبويبات الإعدادات:
  - هوية الفندق
  - معلومات التواصل
  - إعدادات التشغيل
  - المطعم والكافتريا
  - السياسات
  - الفواتير
- المطلوب كان تغيير **خلفية الزر عند الوقوف عليه** وليس لون النص فقط.

### 3. زر العين الخاص بكلمة المرور
- تم إصلاح زر إظهار/إخفاء كلمة المرور بشكل مركزي في كل المشروع.
- يجب أن يبقى الزر بلونه الأساسي الهادئ، ولا يرجع للتدرج أو الظل المزعج السابق.
- لا يجب أن يؤثر هذا الإصلاح على خلفيات الـ Sidebar أو Topbar أو المحتوى.

### 4. الخلفيات المتدرجة العامة
- تم استعادة الخلفيات المتدرجة للـ Sidebar والـ Topbar وخلفية المحتوى.
- يجب الحفاظ على الإحساس البصري السابق: خلفيات ناعمة، زجاجية، ومتدرجة، وليست بيضاء مسطحة.

## قبل تسليم أي نسخة جديدة

يجب التأكد من النقاط التالية:

- الخلفية المتدرجة للـ Sidebar ظاهرة.
- الخلفية المتدرجة للـ Topbar ظاهرة.
- خلفية منطقة المحتوى ما زالت ناعمة ومتدرجة.
- Hover تبويبات الإعدادات ما زال يغير الخلفية.
- زر الحذف الصغير واضح عند Hover.
- زر العين في حقول كلمة المرور واضح وأنيق بكل الصفحات والمودالات.
- تشغيل:

```powershell
npm run check
npm run smoke:test
```

ثم تحديث هذا الملف إذا أضيف تعديل جديد.
