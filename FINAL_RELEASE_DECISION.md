## 2) نوع الإطلاق المسموح — Controlled Launch / إطلاق متحكّم

- البدء بعددٍ **محدود من الفنادق** (دفعة أولى صغيرة موثوقة).
- **مراقبة يومية** في الأسبوع الأول (logs / أخطاء 5xx / الحجوزات / المدفوعات).
- **عدم فتح التسجيل العام العشوائي** للفنادق مباشرةً — الإضافة عبر صاحب المنصّة أوّلًا.
- **نسخ احتياطي (backup) مُفعَّل** قبل استقبال أيّ بيانات حقيقية.
- **مراقبة (monitoring) + تنبيهات** على التوفّر وأخطاء الخادم.
- **مراجعة أول أسبوع تشغيل** قبل أيّ توسّع.

> ⚠️ **لا** يُوصى بتوسّع ضخم أو تسجيل مفتوح بلا مراقبة. الجاهزية الحالية = تقنيّة؛ تحتاج تأكيدًا على بيانات إنتاجية أولية.

## 6) ما الذي أُغلق (سلسلة التحصين 1–13)

1. **FolioCharge Financial Safety** — منع الحذف واستبداله بـ void (سبب + Audit + استثناء من الرصيد).
2. **Public Booking Eligibility Gate** — توحيد ظهور/حجز الفندق في `eligibility.py` (حالة/اشتراك/باقة/اتفاقية/غرف)، القرار خادميّ.
3. **Public Booking Manage Token Hardening** — `manage_token`/`manage_url` مرّة واحدة عند الإنشاء؛ لا كشف في البحث.
4. **Split Payment Consistency (Payment)** — مجموع طرق الدفع = المبلغ، بلا سالب، Decimal.
5. **Public Booking Phone Normalization** — حقل مطبّع مفهرس + backfill + مقارنة مطبّعة (لا يُكشف في العام).
6. **Permission Role Fallback Hardening** — الدور من `UserProfile` فقط، fail‑closed، لا استنتاج من الاسم.
7. **Public API Throttling** — scope مستقلّ لكل نقطة عامة حسّاسة (توفّر/حجز/بحث/إلغاء/تقييم).
8. **FoodOrder Split Payment Consistency** — مجموع الأجزاء الأربعة = الإجمالي، بلا سالب.
9. **Production Settings & `check --deploy`** — إعدادات إنتاج من env بقيَم آمنة؛ `check --deploy` = 0 issues.
10. **Documentation Alignment** — توثيق مطابق للواقع، بلا ادّعاء جاهزية مبكّرة.
11. **Full Test Suite & Frontend Build** — 216/216 + فحوص الواجهة.
12. **Full E2E Operational Scenario** — سيناريو تشغيليّ 8/8 + حالات سلبية.
13. **Frontend Build Fix** — `experimental: { cpus: 4 }` أوقف تعليق البناء (EPIPE) على الأجهزة كثيرة الأنوية.

## 8) شروط النشر التشغيلية (إلزامية قبل استقبال بيانات حقيقية)

- **PostgreSQL** إنتاجيّ عبر `DATABASE_URL` (لا SQLite).
- `DJANGO_DEBUG=false`.
- `DJANGO_SECRET_KEY` قويّ (الإقلاع يفشل بدونه).
- `DJANGO_ALLOWED_HOSTS` مضبوط للدومين.
- `CORS_ALLOWED_ORIGINS` + `CSRF_TRUSTED_ORIGINS` مضبوطان (لا CORS مفتوح).
- **HTTPS** مفعّل + **HSTS** بعناية + **Secure cookies** (تلقائيًّا عند `DEBUG=false`).
- `python manage.py migrate`.
- `python manage.py collectstatic --noinput`.
- `python manage.py create_platform_owner` (مالك المنصّة الرسميّ بأمان).
- ضبط **domain** + reverse proxy يمرّر `X‑Forwarded‑Proto`/`X‑Forwarded‑For`.
- ضبط **backup** + **logs** + **monitoring**.
- **Smoke test بعد النشر** (القسم 9).
